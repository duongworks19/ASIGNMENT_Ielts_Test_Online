Use Case ID: UC_LOGIN
Use Case Name: User Login
Created By: [Tên của bạn]
Date Created: 13/06/2026
Actors: IELTS User (Student, Tutor, Admin)
Triggers: User clicks on "Login" button
Description: User logs into the system to access their respective dashboard and IELTS content
Pre-conditions: User has a registered account in the system
Post-conditions: User is successfully authenticated and redirected to their dashboard
Normal Flow: UC_LOGIN_NF_1. User clicks on login.
UC_LOGIN_NF_2. The system displays the login form.
UC_LOGIN_NF_3. User fills up Email (Registered IELTS ID) and Password.
UC_LOGIN_NF_4. User submits the form.
UC_LOGIN_NF_5. The system validates the input data:
    - User email is found in the registered student/tutor/admin database.
    - Entered password matches the encrypted password for the validated email ID.
    - Account status (e.g., Active) allows access to the IELTS content.
UC_LOGIN_NF_6. All validation success. The system initiates User Dashboard Access (IELTS Content & Tests).
Alternative Flows: UC_LOGIN_AT_1. At step UC_LOGIN_NF_5, if email is not found or password does not match, the system shows an error message.
UC_LOGIN_AT_2. At step UC_LOGIN_NF_5, if the account status is not Active, system denies access and shows error message.
Exceptions: System error or database connection error
Priority: High
Other information: Email must be valid. Required fields cannot be empty.
Frequency of Use: Usually
Business Rules: Account status must allow access (Student, Tutor, Admin dashboards)
Assumptions: Database is accessible and functioning
